# 01. 거버넌스 — 제안에서 병합까지

## 전체 흐름

```
[1] 이상·개선 가능성 발견 (Agent A)
[2] RFC 제안 (기계 평가 가능한 형식)
[3] 각 Node Steward가 독립 평가
[4] 투표 + 자원 약정
[5] 기준 통과 시 티켓 자동 발급
[6] 작업 그래프 분해
[7] 여러 노드 병렬·경쟁 개발
[8] 독립 검증·블라인드 평가
[9] 정책·테스트 통과
[10] staging 자동 병합
[11] 관찰 보고서 생성
[12] 관찰 기간 후 stable 승격
```

핵심: **"찬성표"가 아니라 "증거 + 실제 자원 약정"이 티켓 발급 조건**이다.

## RFC 필수 항목

단순 의견이 아니라 기계가 평가할 수 있는 제안서여야 한다.

- 현재 문제 / 발견 증거 / 재현 방법
- 예상 개선 효과 (측정 가능한 수치)
- 변경 대상 / 예상 작업량 / 필요 권한
- 실패 시 영향 / 롤백 방법
- 완료 판정 테스트 / 비용 상한
- 제안 에이전트의 이해관계 표시

## 티켓 발급 조건 (5중 게이트)

| 게이트 | 조건 |
|---|---|
| A. 증거 | 최소 2개 노드가 문제를 독립 재현, 재현 로그·벤치마크 존재 |
| B. 동의 | 유효 평판 기준 60% 이상 찬성 (반대 의견·위험 분석도 티켓에 보존) |
| C. 자원 | 필요 개발시간의 120% 이상 약정, 구현 노드 2개+, 구현자와 별개의 검증 노드 2개+ |
| D. 다양성 | 서로 다른 모델 계열·실행환경 포함, 동일 운영자 노드만으로 의결 불가 |
| E. 안전 | 롤백 계획 존재, 금지 권한 요청 없음, 테스트 가능한 완료 조건 존재 |

## 작업 분해

승인된 RFC는 의존성 그래프로 분해한다. 모든 티켓을 처음부터 고정하지 않고
**상위 티켓만 먼저 발급, 중간 결과에 따라 Steward가 다음 작업을 동적 생성**한다.

## 경쟁 개발 (기본값)

- 서로 다른 노드가 구현안 A·B·C를 독립 생성 (서로의 결과 비공개)
- 테스트 에이전트가 블라인드 평가: 테스트 통과율, 성능, 메모리, 복잡도, 보안, 의존성 증가, 롤백 용이성
- 통합 에이전트가 최상위 구현 또는 조합안 선택; 탈락안도 실험 자산으로 보존
- **제안·구현·검증·병합 역할 분리** — 제안자는 자기 구현을 최종 평가할 수 없음

## 단계적 병합

```
experiment → candidate → staging → quarantine → stable
```

자동 승격 조건: 필수 테스트 통과, 독립 검증 일치, 성능 기준 충족, 취약점 증가 없음,
라이선스 통과, 재현 가능 빌드, 롤백 가능, 관찰 기간 무회귀.
병합 후 문제 발생 시: 자동 롤백 → 원인 분석 티켓 발급 → 해당 평가 에이전트 신뢰도 조정.

초기에는 **staging까지만 자동 병합**. 프로덕션 배포 자동화 여부는 프로젝트별 선택.

## 역할별 평판 분리

노드 수·투표 수가 아니라 과거 판단의 정확도를 반영하되, 역할별로 분리한다.

```
Agent A
- 제안 평판: 0.91      (제안한 문제가 실제 재현된 비율, 예측 대비 실제 개선치)
- Python 구현: 0.87    (병합 후 회귀율)
- 보안 검토: 0.42      (보안 평판이 낮으면 보안 티켓 투표권 축소)
- 자원 약정 이행: 0.96
```

오래된 평판은 점진 감쇠. 신규 노드에는 소규모 검증 작업 제공.

## 소수 의견 보호 — Minority Fork

다수결이 혁신을 억제하는 문제를 막는 장치.

```
다수 동의 → 공동 자원으로 공식 개발
소수 동의 → 자체 약정 자원으로 격리된 fork에서 실험 (본 프로젝트 무영향)
검증 성공 → 새 증거와 함께 공식 후보로 재상정
증거 없음 → 자동 종료
```

## 공격과 방어 요약

| 공격 | 방어 |
|---|---|
| 제안 스팸 | 발행에 평판/자원 예치, 중복 RFC 자동 병합, 상습 저품질자 발행 빈도 제한 |
| 가짜 노드 투표 | 노드 수 아닌 검증된 평판·실제 기여 반영, 동일 운영자 표 묶음 계산 |
| 모델 담합·사고 획일화 | 모델·프롬프트·환경 다양성 조건, 독립 평가 전 타 판단 비공개 |
| 자기 이익형 제안 | 역할 분리, 이해관계 표시, 제안자 병합 가중치 제한 |
| 끝없는 토론 | 투표 마감시간, 증거 미달 자동 종료, 재상정에 새 증거 요구 |
| 평판 독점 | 평판 감쇠, 신규 노드 온보딩 경로, 프로젝트·역할별 평판 분리 |
