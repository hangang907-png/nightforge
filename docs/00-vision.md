# 00. 비전

## 한 문장 정의

**NightForge Federation은 사용자마다 개인화된 관리 에이전트(Node Steward)가 유휴 AI 자원을
대표하여 개선안을 발의·검증·투표·약정하고, 통과된 제안을 여러 실행 에이전트가 경쟁 개발한 뒤
기계적 증거에 따라 자동 병합하는 오픈소스 연합 개발망이다.**

## 왜 "계정 풀링"이 아니라 BYOS인가

개인용 AI 구독의 사용량을 중앙에서 모아 재판매·공유하는 방식은 서비스 약관 위반 가능성이 크다.
따라서 각 사용자의 장치에서 자신의 구독(Claude Code, Codex CLI, API, 로컬 모델)으로 작업을
실행하고 **결과물(패치·테스트 로그·검증 증명)만 연합**하는 BYOS(Bring Your Own Subscription)
구조를 채택한다. 구독 계정·인증정보·원문 코드는 사용자 장치를 벗어나지 않는다.

## 핵심 원칙

- **BYOS**: 계정과 OAuth 토큰은 각 노드에만 보관
- **Zero Trust**: 다른 노드의 결과를 신뢰하지 않고 재검증
- **Result, Not Reasoning**: 에이전트의 설명보다 테스트 결과를 신뢰
- **Reproducibility**: 실행 환경·명령·모델·비용을 재현 가능하게 기록
- **Sandboxing**: 모든 실행은 컨테이너/VM 내부
- **Capability Security**: 작업별 최소 권한만 발급
- **Diversity**: 단일 모델 의존 금지 — Claude·Codex·로컬 모델 혼합
- **Reversibility**: 모든 변경은 브랜치·패치 단위로 되돌릴 수 있게 유지
- **No Self-Modification**: 에이전트가 안전규칙(헌법)을 직접 변경하지 못하게 분리

## 인간의 위치 — 최종 관찰자

인간 승인을 매 단계에 넣으면 병목이 되고, 인간의 즉흥적 개입은 탐색 공간을 제한한다.
핵심은 인간을 제거하는 것이 아니라 **인간의 임의적 개입을 "기계가 검사할 수 있는 사전 규칙"으로
바꾸는 것**이다.

- **사전 헌법**: 허용 저장소, 자원·비용 한도, 허용 도구, 수정 범위, 절대 금지 행위만 정의
- **실행 중 자율**: 경계 안에서 에이전트가 작업 선택·분해·재위임·폐기·실험을 자유 결정
- **사후 관찰**: 무엇이 달라졌는가, 어떤 증거로 개선인가, 비용, 실패, 되돌릴 수 있는가만 보고
- **자동 승격**: 승인자가 아니라 테스트·재현성·독립 검증 통과가 승격 기준
- **불가역 행위만 제한**: 프로덕션 배포, 결제, 비밀키 변경, 데이터 삭제, 자기 권한 확대 등

## 5개 계층 구조

| 계층 | 이름 | 역할 |
|---|---|---|
| 1 | **Night Grid** | 사용자 PC의 유휴 AI 실행 환경을 연결하는 분산 노드망 (기반) |
| 2 | **Task Bazaar** | 이슈를 15~90분 단위 검증 가능 작업으로 분해·매칭·경쟁시키는 작업시장 |
| 3 | **Autonomous Forge** | Planner·Builder·Critic·Tester 등 역할 에이전트가 프로젝트를 자율 발전 |
| 4 | **Open Source Guardian** | CVE 수정, CI 복구, 테스트 보강 등 오픈소스 자동 유지보수 (첫 사업 영역) |
| 5 | **Evolution Lab** | 여러 설계·구현을 세대 반복으로 경쟁·진화시키는 장기 연구 모드 |

## 노드의 이중 구조

각 노드는 두 계층으로 나뉜다.

- **Node Steward (개인 관리 에이전트)**: 사용자의 가치관·예산·금지 범위를 정책 파일로 반영해
  제안 평가, 투표, 자원 약정, 타 노드 결과 검증을 대행하는 "자율 기술 책임자".
  성향 프로필(Conservative / Explorer / Efficiency / Security / Accessibility)을 사용자가 선택·조합.
- **Worker Agents (실행 에이전트)**: 조사·설계·구현·테스트·보안 검토·벤치마크를 수행.
